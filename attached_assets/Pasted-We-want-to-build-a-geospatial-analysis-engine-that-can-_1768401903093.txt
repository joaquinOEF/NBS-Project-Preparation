We want to build a geospatial analysis engine that can, on request, ingest a set of global, open datasets and derive a consistent stack of climate-relevant metrics for any city. These datasets include elevation (Copernicus DEM -we've already integrated this one), land cover and imperviousness (ESA WorldCover / MapBiomas), hydrography and surface water (HydroSHEDS, OSM waterways, JRC Global Surface Water), vegetation and canopy (Hansen), population exposure (WorldPop), and optional validation layers (observed floods, land surface temperature). Rather than performing high-resolution modeling, the system standardizes all inputs onto a coarser, city-scale working grid (e.g. 250–500 m), ensuring computational efficiency, global applicability, and interpretability.
On top of this standardized grid, the engine computes a minimal but meaningful set of derived indicators—such as slope, flow accumulation, impervious cover, vegetation cover, canopy density, river proximity, and population exposure—and normalizes them using within-city percentiles. These indicators are then combined into transparent composite scores for flooding, heat, and landslide risk, allowing each grid cell to be classified by its dominant hazard or as a combined-risk area. Adjacent cells with similar hazard profiles are merged into a single, mutually exclusive and collectively exhaustive (MECE) set of planning units that divide the city into manageable sections, similar in spirit to how cities like Copenhagen package interventions by catchment or district.
To ensure the system is fast, reproducible, and scalable, all intermediate and final outputs—including clipped rasters, per-cell metrics, composite scores, and aggregated planning units—are cached and stored in a database or file-based cache keyed by city and analysis parameters. This avoids recalculating expensive terrain and zonal statistics for repeated requests and allows the platform to serve results instantly after the first run. The end goal is a reusable, data-driven foundation that supports multi-hazard planning, scenario simulation, and nature-based solution prioritization without requiring heavy modeling or city-specific datasets.
