We want to turn the current project builder into a **persistent, agent-aware knowledge workspace** for preparing Nature-Based Solution projects at pre-feasibility stage. Instead of treating each Info Block (Funder Selection, Site Explorer, Impact Model, Project Operations, Business Model) as a linear form or a one-off export step, the UI evolves into a set of **interconnected modules** that can be filled independently and revisited over time. Each block exposes structured fields, contextual explanations, and AI-assisted actions (autocomplete, “suggest”, “draft”, “explain”), and a zone-based map workflow allows users to explore sites, select assets, and attach interventions. A double-click or focused interaction on a zone opens a decision context where the user sees why that zone matters, what intervention categories apply, and which concrete actions are feasible there. Throughout the UI, users can review confidence levels, assumptions, and evidence behind any AI-generated content, ensuring human oversight and validation.

On the backend, we want a **single canonical project state** that both the UI and conversational agents read from and write to. This is implemented as a relational system of record (Postgres) with typed project fields and JSON-based block states, complemented by an evidence layer and an intervention planning layer. A retrieval system (RAG) sits alongside this store, drawing from a curated library of real NBS projects, reference playbooks, and the project’s own evolving content to support agentic reasoning. All AI interactions—whether conversational or triggered from UI buttons—go through the same write/patch interface, producing auditable, field-level updates linked to evidence and assumptions. Optional graph projections can be derived from this core data to support cross-project reasoning and recommendations, but the authoritative structure remains simple, legible, and stable, enabling the platform to generate credible concept notes while staying flexible about output formats and future workflows.
